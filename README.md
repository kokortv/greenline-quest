# Квест отеля с QR-кодами

Мобильная игра для гостей отеля. Гость один раз регистрируется по основному QR-коду, видит карту, ищет QR-коды персонажей, отвечает на загадки и получает итоговую карточку с именем, комнатой и баллами.

## QR-ссылки

Основной QR ведет на:

- `index.html`

QR персонажей ведут на:

- `index.html?spot=lobby`
- `index.html?spot=cafe`
- `index.html?spot=pool`
- `index.html?spot=garden`
- `index.html?spot=stairs`

Откройте `qr-links.html`, вставьте публичный адрес игры и получите готовые ссылки для QR-генератора.

## Механика гостя

1. Гость вводит имя и номер комнаты.
2. Имя проверяется на цифры, странные символы и слова из стоп-листа.
3. Комната должна быть в списке существующих номеров.
4. После регистрации открывается карта.
5. На карте видны только доступные по погоде/времени точки.
6. До сканирования точка серая, без имени персонажа.
7. После сканирования QR персонажа начисляются баллы за находку.
8. На загадку дается 3 попытки.
9. Бонус за ответ зависит от попытки: больше за первую, меньше за вторую и третью.
10. После правильного ответа или после 3 неудач гость возвращается на карту.
11. Когда все доступные персонажи найдены, показывается финал и карточка результата для сохранения.
12. На этом же устройстве нельзя начать заново: состояние хранится в браузере.

## Админка

Админ-страница:

- `admin.html`

В ней можно менять:

- текущую погоду квеста: `sun`, `rain`, `cloudy`;
- список комнат;
- имена персонажей;
- места;
- координаты на карте;
- загадки и ответы;
- баллы за находку;
- баллы за 1/2/3 попытку;
- правила доступности по погоде и времени.

`any` используется только у персонажа и означает “доступен при любой погоде”. В общих настройках погода должна быть конкретной: `sun`, `rain` или `cloudy`.

## Подключение Google Sheets

1. Создайте Google Sheet.
2. Откройте `Расширения -> Apps Script`.
3. Вставьте код из `google-apps-script.js`.
4. Нажмите `Deploy -> New deployment -> Web app`.
5. Скопируйте Web app URL.
6. В `config.js` вставьте URL в поле `sheetEndpoint`.

Apps Script создает листы:

- `Config` — JSON с настройками, комнатами и персонажами.
- `Events` — события игроков: регистрация, находка, ответы, завершение.

Если `sheetEndpoint` пустой, игра и админка работают в демо-режиме через `config.js` и `localStorage`.
